---
title: 'GET /api/packages — List and Filter Packages'
sidebarTitle: 'List Packages'
description: 'Retrieve a paginated list of packages in your warehouse. Filter by status, courier, tracking number, or intake date range.'
---

Use this endpoint to query all package records associated with your warehouse account. You can narrow results with filters for shipment status, courier partner, tracking number substring, or a date range based on when packages were first logged. Responses are paginated and sorted by creation date descending.

## Request

<ParamField header="X-API-KEY" type="string" required>
  Your Shiipp API key. Pass this in the request header to authenticate.
</ParamField>

**Method:** `GET`  
**URL:** `/api/packages`

### Query Parameters

<ParamField query="page" type="integer" default={1}>
  The page number to retrieve. Starts at `1`.
</ParamField>

<ParamField query="per_page" type="integer" default={25}>
  Number of results per page. Maximum value is `100`.
</ParamField>

<ParamField query="status" type="string">
  Filter packages by their current status. Accepted values: `pending`, `in_transit`, `arrived`, `delivered`, `exception`.
</ParamField>

<ParamField query="courier_id" type="string">
  Filter results to packages handled by a specific courier partner. Use the `id` value returned from `GET /api/couriers`.
</ParamField>

<ParamField query="tracking" type="string">
  Perform a partial match search on tracking numbers. For example, `1Z999` will match all tracking numbers that contain that substring.
</ParamField>

<ParamField query="from_date" type="string">
  Start of the intake date range. Must be an ISO 8601 date string (e.g. `2024-01-01`). Inclusive.
</ParamField>

<ParamField query="to_date" type="string">
  End of the intake date range. Must be an ISO 8601 date string (e.g. `2024-01-31`). Inclusive.
</ParamField>

### Example Request

<CodeGroup>
```bash cURL
curl --request GET \
  --url "https://api.shiipp.com/api/packages?status=arrived&per_page=25&page=1" \
  --header "X-API-KEY: your_api_key_here"
```
</CodeGroup>

## Response

<ResponseField name="success" type="boolean">
  Indicates whether the request was processed successfully.
</ResponseField>

<ResponseField name="data" type="array">
  An array of package objects matching the applied filters.

  <Expandable title="Package object fields">
    <ResponseField name="id" type="string">
      Unique package identifier (e.g. `pkg_01HZ3K5MNPQR2STV`).
    </ResponseField>
    <ResponseField name="tracking" type="string">
      The carrier-assigned tracking number.
    </ResponseField>
    <ResponseField name="customer_name" type="string">
      Full name of the package recipient.
    </ResponseField>
    <ResponseField name="courier" type="string">
      Display name of the courier partner assigned to this package.
    </ResponseField>
    <ResponseField name="weight_lbs" type="number">
      Package weight in pounds.
    </ResponseField>
    <ResponseField name="weight_kg" type="number">
      Package weight in kilograms.
    </ResponseField>
    <ResponseField name="status" type="string">
      Current package status. One of: `pending`, `in_transit`, `arrived`, `delivered`, `exception`.
    </ResponseField>
    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp of when the package record was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta" type="object">
  Pagination metadata for the current response.

  <Expandable title="Meta fields">
    <ResponseField name="total" type="integer">
      Total number of packages matching the applied filters.
    </ResponseField>
    <ResponseField name="page" type="integer">
      The current page number returned.
    </ResponseField>
    <ResponseField name="per_page" type="integer">
      Number of results per page for this response.
    </ResponseField>
  </Expandable>
</ResponseField>

</Note>

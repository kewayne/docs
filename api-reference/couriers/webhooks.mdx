---
title: 'POST /api/couriers/{id}/webhooks — Register Webhooks'
sidebarTitle: 'Register Webhooks'
description: 'Register an HTTPS webhook endpoint for a courier partner to receive real-time event notifications for package and customer activity.'
---

Use this endpoint to register a webhook listener on a specific courier partner. Once registered, Shiipp will send an HTTP `POST` request to your target URL whenever a subscribed event occurs — such as a new package intake, a status change, or a customer sync. You can register multiple webhooks per courier, each with its own URL and event subscription list.

## Request

<ParamField header="X-API-KEY" type="string" required>
  Your Shiipp API key. Pass this in the request header to authenticate.
</ParamField>

**Method:** `POST`  
**URL:** `/api/couriers/{id}/webhooks`  
**Content-Type:** `application/json`

### Path Parameter

<ParamField path="id" type="string" required>
  The unique identifier of the courier partner to attach this webhook to (e.g. `cou_01HZ`). Use `GET /api/couriers` to retrieve valid IDs.
</ParamField>

### Body Parameters

<ParamField body="url" type="string" required>
  The HTTPS endpoint URL that will receive webhook payloads. Must use `https://` — plain HTTP URLs are rejected.
</ParamField>

<ParamField body="events" type="string[]" required>
  An array of event types to subscribe to. At least one event is required. Accepted values:
  - `customer_sync` — fired when a customer record is synchronized
  - `package_intake` — fired when a new package is logged for this courier
  - `status_update` — fired when a package status changes
</ParamField>

<ParamField body="active" type="boolean" default={true}>
  Whether this webhook should begin receiving events immediately after registration. Set to `false` to register the webhook in a paused state.
</ParamField>

### Example Request

<CodeGroup>
```bash cURL
curl --request POST \
  --url "https://api.shiipp.com/api/couriers/cou_01HZ/webhooks" \
  --header "X-API-KEY: your_api_key_here" \
  --header "Content-Type: application/json" \
  --data '{
    "url": "https://your-system.com/webhooks/shiipp",
    "events": ["package_intake", "status_update"],
    "active": true
  }'
```

```json Request Body
{
  "url": "https://your-system.com/webhooks/shiipp",
  "events": ["package_intake", "status_update"],
  "active": true
}
```
</CodeGroup>

## Response

A successful request returns HTTP `201 Created` along with the new webhook object, including a signing secret.

<ResponseField name="success" type="boolean">
  `true` when the webhook was registered successfully.
</ResponseField>

<ResponseField name="data" type="object">
  The newly created webhook registration object.

  <Expandable title="Webhook object fields">
    <ResponseField name="id" type="string">
      Unique identifier for this webhook registration (e.g. `wh_01HZ`).
    </ResponseField>
    <ResponseField name="url" type="string">
      The HTTPS endpoint URL registered to receive payloads.
    </ResponseField>
    <ResponseField name="events" type="array">
      The list of event types this webhook is subscribed to.
    </ResponseField>
    <ResponseField name="secret" type="string">
      A `whsec_`-prefixed signing secret used to verify that incoming webhook payloads originate from Shiipp. **Returned only on creation.**
    </ResponseField>
    <ResponseField name="active" type="boolean">
      Whether this webhook is currently active and receiving events.
    </ResponseField>
    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp of when the webhook was registered.
    </ResponseField>
  </Expandable>
</ResponseField>

```json Example Response (201 Created)
{
  "success": true,
  "data": {
    "id": "wh_01HZ",
    "url": "https://your-system.com/webhooks/shiipp",
    "events": ["package_intake", "status_update"],
    "secret": "whsec_abc123...",
    "active": true,
    "created_at": "2024-01-15T10:30:00Z"
  }
}
```

<Warning>
  The `secret` field is only returned once, at the moment of creation. Store it immediately in a secure location such as an environment variable or secrets manager. If you lose the secret, you will need to delete and re-register the webhook to obtain a new one.
</Warning>

<Tip>
  The signing secret is used to verify webhook authenticity by validating the `X-Shiipp-Signature` header included with every delivery. To configure what data is included in webhook payloads for each event type, see the [Webhooks integration guide](/integrations/webhooks).
</Tip>

## Webhook Payload Variables

Shiipp interpolates the following variables into webhook payload templates when delivering events. Use these placeholders in your payload template configuration:

| Variable | Description |
|----------|-------------|
| `{{package_id}}` | Unique identifier of the package that triggered the event. |
| `{{tracking}}` | Carrier tracking number of the package. |
| `{{weight_lbs}}` | Package weight in pounds. |
| `{{weight_kg}}` | Package weight in kilograms. |
| `{{status}}` | Current package status at the time of the event. |
| `{{courier}}` | Display name of the courier partner. |
| `{{customer_name}}` | Full name of the package recipient. |
